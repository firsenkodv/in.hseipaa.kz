Функционал «SEO»

Централизованное управление metatitle / description / keywords для всех моделей сайта

1. Обзор

Цель: SEO-ресурс в MoonShine всегда пуст — это значит, что SEO заполнено на всех страницах.

Таблица seos является зеркалом SEO-полей из моделей. Администратор заполняет данные через SEO-ресурс — они записываются напрямую в модель (metatitle, description, keywords). Ресурс показывает только те записи, где хотя бы одно поле пустое.

Сохранение модели
→
syncSeoRecord()
→
копия полей в seos
SEO-ресурс (MoonShine)
→
afterUpdated()
→
запись обратно в модель

Статические страницы (home, contacts и др.) управляются через настройки MoonShine и в SEO-ресурсе не отображаются.

Охват моделей

УровеньМодели
РазделыService, Useful
КатегорииSiteNew, ServiceCategory, CompanyCategory, UsefulCategory, UsefulSubcategory
МатериалыSiteNewItem, UsefulItem, ServiceItem, CompanyItem, Tax, Mzp

2. База данных

Таблица seos

КолонкаТипОписание
idbigint PK—
keyvarchar UNIQUEСоставной идентификатор: {prefix}:{id} или page:{slug}
model_classvarchar nullableFQCN модели (App\Models\SiteNewItem). NULL для статических страниц
labelvarcharЧитаемое название (обновляется автоматически из title модели)
titlevarchar nullableЗеркало поля metatitle модели
descriptiontext nullableЗеркало поля description модели
keywordstext nullableЗеркало поля keywords модели

Формат ключа (key)

Ключ = префикс из конфига + : + ID записи:

site_new_item:1      →  SiteNewItem с id=1
service_category:5   →  ServiceCategory с id=5
tax:3                →  Tax с id=3
page:home            →  статическая страница (model_class = null)

По ключу afterUpdated() определяет модель для обратной записи: парсит id → делает $item->model_class::find($id).

3. Модель

App\Models\Seo

// Только незаполненные модельные записи (используется в SeoResource)
Seo::incomplete()->get();

// Поиск по ключу — только для статических страниц
seo_override('page:home');

Скоуп incomplete возвращает записи, где model_class IS NOT NULL и хотя бы одно из title / description / keywords — пустое или NULL.

4. Трейт HasSeo

App\Models\Concerns\HasSeo — подключён к 13 моделям:

use App\Models\Concerns\HasSeo;

class SiteNewItem extends Model
{
    use HasSeo;
}
МетодОписание
bootHasSeo()Слушает событие saved, вызывает syncSeoRecord(). Обёрнут в try/catch — не ломает сохранение если таблица не создана
syncSeoRecord()Создаёт или обновляет запись в seos: копирует metatitle→title, description→description, keywords→keywords, заполняет model_class и label
seoPrefix(): stringОпределяет префикс ключа: обратный поиск по классу в config/seo.models
seoLabel(): stringВозвращает $this->title для поля label в таблице seos

6. Хелпер seo_override()

src/Support/Helpers/helpers.php

function seo_override(string $key): ?Seo

Используется только для статических страниц (где нет Eloquent-модели). Возвращает запись Seo или null.

7. Компонент x-seo.meta

resources/views/components/seo/meta.blade.php

Модельные страницы — без seo-key

SEO берётся напрямую из полей модели через пропы. seo-key не передаётся:

<x-seo.meta
    title="{{ ($item->metatitle) ?? $item->title }}"
    description="{{ $item->description }}"
    keywords="{{ $item->keywords }}"
/>

Статические страницы — с seo-key

Передаётся seo-key; компонент вызывает seo_override() и перекрывает пропы если в seos есть данные:

<x-seo.meta
    title="{!! config2('moonshine.home.metatitle') !!}"
    description="{!! config2('moonshine.home.description') !!}"
    keywords="{!! config2('moonshine.home.keywords') !!}"
    seo-key="page:home"
/>
Вызов seo_override() внутри компонента обёрнут в try/catch — если таблица seos ещё не создана, страница не ломается и использует значения из пропов.

8. Администратор (MoonShine)

Ресурс: app/MoonShine/Resources/SeoResource.php

Меню: MenuItem::make('SEO', SeoResource::class, 'magnifying-glass')

Что показывает ресурс

Только записи с model_class IS NOT NULL, где хотя бы одно поле пустое. Сортировка: сначала записи с максимальным числом пустых полей.

Когда ресурс пуст — SEO заполнено на всех страницах сайта.

Поля формы редактирования

ПолеРедактируетсяПримечание
Страница / Материал (label)нетОбновляется автоматически при сохранении модели
Ключ (key)нетФормат prefix:id
TitleдаПишется в seos.title и в model.metatitle
DescriptionдаПишется в seos.description и в model.description
KeywordsдаПишется в seos.keywords и в model.keywords

Обратная запись в модель (afterUpdated)

// SeoResource::afterUpdated()
$id = last(explode(':', $item->key));           // извлекаем id из ключа
$model = $item->model_class::find($id);         // находим модель
$item->model_class::withoutEvents(function() {  // без повторного события saved
    $model->update([
        'metatitle'   => $item->title,
        'description' => $item->description,
        'keywords'    => $item->keywords,
    ]);
});

Ограничения действий

Отключены: CREATE VIEW DELETE

9. Первоначальное заполнение

php artisan migrate
php artisan seo:seed

Что делает seo:seed:

  1. Проходит по всем записям 13 моделей из config/seo.models
  2. Для каждой вызывает $item->syncSeoRecord() — копирует текущие metatitle / description / keywords в seos
  3. Создаёт заглушки для статических страниц (model_class = null) — они не попадают в SEO-ресурс
Команда идемпотентна — повторный запуск обновляет существующие записи без дубликатов.
Новые записи появляются в seos автоматически при первом сохранении через событие saved.

10. Как подключить новую модель

1. Убедиться, что в модели есть поля metatitle, description, keywords.

2. Добавить трейт:

use App\Models\Concerns\HasSeo;

class NewModel extends Model
{
    use HasSeo;
}

3. Добавить префикс в config/seo.php → models:

'new_prefix' => \App\Models\NewModel::class,

4. View менять не нужно — SEO берётся из полей модели напрямую:

<x-seo.meta
    title="{{ ($item->metatitle) ?? $item->title }}"
    description="{{ $item->description }}"
    keywords="{{ $item->keywords }}"
/>

5. Запустить php artisan seo:seed для существующих записей.
