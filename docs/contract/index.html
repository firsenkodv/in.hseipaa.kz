Договоры — полная цепочка

Модуль создания, рассылки и подписания договоров на обучение. Модель: App\Models\Contract · Таблица: contracts

Общее описание

Договор создаётся администратором через модальное окно в кабинете. После создания пользователь получает письмо со ссылкой на публичную страницу, где он может ознакомиться с условиями и подписать договор. Подписание фиксирует IP и время. После подписания договор заблокирован от изменений.

Кто создаёт
Администратор кабинета через модальное окно. Требуется сессионный ключ session('a').
Кто подписывает
Пользователь — двумя способами: по публичной ссылке (без авторизации) или из личного кабинета.
Хранение
Таблица contracts. Публичный токен (64 символа) — доступ без авторизации.
Нумерация
Формат NN/MM/YY — порядковый номер в году / месяц / год. Генерируется в контроллере при создании.

Полная цепочка создания и подписания

🖱️
1. Открытие модального окна создания договора
Администратор нажимает кнопку на странице пользователя. Кнопка имеет атрибуты data-form="admin_contract_create" и data-transfer='{"user_id": N}'. JavaScript (fancybox.js) перехватывает клик, отправляет POST на /fancybox-ajax с шаблоном admin_contract_create и данными пользователя.

FancyBoxController::fancybox() получает запрос, загружает:
  • Объект пользователя (User::findOrFail($userId))
  • Список дисциплин из таблицы trainings (сортировка по sorting desc)
  • Список валют из config('currency.currency'), дефолт — KZT
  • Список организаций из OrganizationEnum::cases()
Возвращает HTML модального окна, Fancybox отображает его. Инициализируются: custom select, IMask (телефон, цена, часы), датапикер периода.

2. Заполнение формы и создание договора
Администратор заполняет поля формы. JavaScript (async_execution.js) собирает данные:
  • Из .app_input_name-полей: user_id, fio, email, phone, date_from, date_to, price, hours
  • Из .app_selected-элементов (custom select): training_id, organization, currency
Отправляет JSON через Axios на POST /admin_contract_create.

AdminContractCreateRequest валидирует данные. В prepareForValidation(): символ * удаляется из training_id и currency, пробелы — из price.

FancyBoxSendingFromFormController::fancyboxAdminContractCreate():
  • Генерирует public_token — Str::random(64)
  • Вычисляет contract_number: NN/MM/YY (макс. порядковый номер по году + 1)
  • Создаёт запись Contract::create([...])
  • Формирует $contractData для письма

Алгоритм генерации номера договора

// Берём максимальный порядковый номер за текущий год
$lastSeq = Contract::whereYear('created_at', now()->year)
    ->whereNotNull('contract_number')
    ->get()
    ->map(fn($c) => (int) explode('/', $c->contract_number)[0])
    ->max() ?? 0;

$contractNumber = sprintf('%02d/%s/%s',
    $lastSeq + 1,   // NN — порядковый номер
    now()->format('m'), // MM — месяц
    now()->format('y')  // YY — год (2 цифры)
);
// Пример: "03/06/26"

3. Email пользователю с ссылкой на договор
После создания записи в БД, контроллер формирует массив данных $contractData и ставит письмо в очередь (Mail::to(...)->queue()).

ContractCreatedMail отправляет HTML-письмо. В письме:
  • Логотип организации с корректными размерами для email-клиентов
  • Таблица с деталями договора (дисциплина, период, стоимость, часы)
  • CTA-кнопка «Открыть и подписать договор» → публичная ссылка
  • Контакты менеджера (если привязан)
Публичная ссылка строится через: route('contract.public', $token) 
// Данные письма
$contractData = array_merge(
    $request->only(['fio', 'email', 'phone', 'training_id', ...]),
    [
        'contract_number'    => $contractNumber,
        'organization_label' => $request->input('organization'),
        'organization_logo'  => $orgEnum ? asset($orgEnum->logo()) : null,
        'manager_name'       => $manager?->username,
        'manager_phone'      => $manager?->phone ? format_phone($manager->phone) : null,
        'public_url'         => $publicUrl,
    ]
);
Mail::to($request->input('email'))->queue(new ContractCreatedMail($contractData));

4. Пользователь открывает договор по публичной ссылке
Ссылка вида https://domain/contract/{token} — без авторизации.

PublicContractController::show($token):
  • Находит договор по public_token
  • Если договор не найден — 404
  • Возвращает чистую HTML-страницу (без layout сайта)
Страница показывает: статус договора (is_signed), полные данные, кнопку «Подписать» (если ещё не подписан).

Также пользователь может открыть и подписать договор из личного кабинета (/cabinet/contracts) через модальное окно.

5. Подписание договора
Существует два способа подписать договор:

Способ A — По публичной ссылке (без авторизации)

Пользователь нажимает «Подписать» на странице /contract/{token}. Страница отправляет AJAX-запрос POST /contract/{token}/sign.

PublicContractController::sign($token):
  • Находит договор по токену, где is_signed = false
  • Устанавливает is_signed = true
  • Захватывает IP пользователя
  • Ставит в очередь ContractSignedMail
  • Возвращает JSON {signed: true}

Способ B — Из личного кабинета (авторизованный пользователь)

Пользователь в кабинете /cabinet/contracts открывает модалку подписания. Форма отправляет contract_id на POST /cabinet_contract_sign.

ContractSignRequest проверяет: exists:contracts,id,user_id,{auth()->id()},is_signed,0 — договор принадлежит этому пользователю и ещё не подписан.

FancyBoxSendingFromFormController::fancyboxContractSign():
  • Загружает договор с отношением user.Manager
  • Устанавливает is_signed = true
  • Ставит в очередь ContractSignedMail
  • Возвращает JSON {contract_id, signed: true}
Важно: После установки is_signed = true договор нельзя редактировать ни через кабинет администратора, ни через MoonShine. Это проверяется на уровне AdminContractUpdateRequest: правило exists:contracts,id,is_signed,0.

6. После подписания — уведомление и обновление UI
ContractSignedMail — письмо-подтверждение. Содержит:
  • Логотип организации
  • Заголовок «Договор подписан On-line»
  • Таблицу с деталями договора
  • Блок «Документ подписан электронной подписью» с временем и IP
  • Раздел «Преимущества обучения»
  • Контакты менеджера и футер

В кабинете администратора UI обновляется без перезагрузки страницы: JavaScript слушает событие form:success, при data.signed === true добавляет CSS-класс user-contracts__item--signed, меняет бейдж с «Ожидает подписания» на «Подписан», убирает кнопку подписи.

При создании договора список договоров пользователя также обновляется через AJAX: GET /cabinet-administrator/ajax/user-contracts/{user_id}. 
// fancybox.js — реакция на подписание
document.addEventListener('form:success', (e) => {

Редактирование договора

Редактирование доступно только для неподписанных договоров. Администратор открывает модальное окно через кнопку на карточке договора (data-form="admin_contract_edit").

AdminContractUpdateRequest содержит правило 'contract_id' => ['required', 'integer', 'exists:contracts,id,is_signed,0'] — если договор подписан, запрос не пройдёт валидацию с ошибкой 422.

Что обновляет fancyboxAdminContractUpdate()

  • ФИО, email, телефон, дисциплину, организацию
  • Даты начала и окончания, стоимость, валюту, часы
  • contract_number и is_signed — не обновляются

Возвращает JSON с обновлёнными данными, UI обновляется без перезагрузки через событие form:success.

app/Http/Requests/CabinetAdmin/AdminContractUpdateRequest.php resources/views/fancybox/forms/cabinet/admin_contract_edit.blade.php

Кабинет пользователя

Страница /cabinet/contracts показывает все договоры текущего пользователя. Для каждого неподписанного договора доступна кнопка «Подписать» — открывает модальное окно cabinet_contract_sign с деталями договора и кнопкой подтверждения.

Компонент для отображения списка: x-cabinet-admin.user.user-contracts с параметрами :signable="true" (пользователь может подписывать) и :editable="false".

MoonShine-панель

Ресурс ContractResource предоставляет полный CRUD для договоров через административную панель MoonShine. Таблица, форма редактирования, детальный просмотр.

Поля в таблице (indexFields)

  • ID (сортируемый), Номер договора, ФИО, Email, Телефон
  • Дисциплина, Цена, Валюта, Часов
  • Дата начала / конца (формат d.m.Y)
  • Подписан — Switcher с updateOnPreview()
  • Пользователь — BelongsTo с кастомным форматированием: username (title)

Методы

  • label() — полное название для отображения
  • logo() — путь к логотипу относительно public/
  • logoSize() — [width, height] для email-шаблонов
  • fromLabel(string $label) — поиск enum по полному названию (используется при создании договора)
  • fromValueSafe(string $value) — безопасный tryFrom()
  • options() — массив [value => label] для select
В БД хранится value enum (too_hse, ip_hse, niise). В форму передаётся label через data-select, затем контроллер преобразует label → value через OrganizationEnum::fromLabel().
