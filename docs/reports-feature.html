Отчёты об обучении

Подача, модерация и переписка по отчётам — полное техническое описание

1. Обзор

Функционал позволяет пользователям подавать отчёты об обучении, а менеджерам — модерировать их. К каждому отчёту привязан отдельный чат между пользователем и его менеджером.

РольВозможности
ПользовательСоздать, редактировать (до принятия), просматривать отчёты, вести переписку с менеджером
МенеджерПросматривать отчёты своих пользователей, редактировать поля, принять отчёт, вести переписку

После принятия ни пользователь, ни менеджер не могут изменять данные отчёта. Чат остаётся доступным.

4. Роль: Пользователь

4.1 Страница отчётов

GET/cabinet/reports — список отчётов пользователя

Для каждого отчёта отображаются период, вид, дисциплина, заведение, файлы сертификатов. Кнопки:

  • «Редактировать» — только при accepted = false
  • «Переписка» — всегда, с красным бейджем 3 числа непрочитанных от менеджера

4.2 Создание отчёта

Открывается через Fancybox (data-form="user_report_create").

POST/user_report_create

Поля: период (датапикер), вид отчёта, дисциплина, заведение, PDF-сертификаты (до 4 файлов, до 15 МБ каждый).

4.3 Редактирование отчёта

Открывается через Fancybox (data-form="user_report_edit", data-transfer='{"report_id": N}'). Доступно только при accepted = false.

POST/user_report_update

4.4 Файлы сертификатов

POST/cabinet.upload.report.files — загрузка (только PDF, макс. 15 МБ, до 4 файлов)
POST/cabinet.delete.report.files — удаление

5. Роль: Менеджер

5.1 Страница отчётов

GET/cabinet-manager/reports — отчёты пользователей менеджера

Для каждого отчёта отображаются имя пользователя, период, данные и статус:

  • Принят или На рассмотрении
  • «Редактировать» — только при accepted = false
  • Иконка чата с красным бейджем 2 — если есть непрочитанные от пользователя

5.2 Просмотр и редактирование (модал)

Открывается через Fancybox (data-form="manager_report_view"). При открытии сообщения от пользователя автоматически помечаются прочитанными.

Состояние отчётаЧто показывается в модале
ПринятУведомление «принят и не может быть изменён», поля read-only, чат доступен
На рассмотренииРедактируемая форма, файлы — только скачивание, кнопки «Сохранить» и «Принять»

5.3 Сохранение изменений

POST/manager_report_update — редактирование полей отчёта

Менеджер может изменить: период, вид отчёта, дисциплину, заведение. Файлы не редактируются.

5.4 Принятие отчёта

POST/manager_report_accept — устанавливает accepted = true

После принятия редактирование заблокировано для обеих сторон навсегда.

6. Чат по отчёту

Каждый отчёт имеет собственную переписку. Сообщения отправляются без перезагрузки страницы (AJAX) и сразу добавляются в DOM.

6.1 Со стороны менеджера

Расположен в нижней части модала manager_report_view. Доступен всегда, в том числе для принятых отчётов.

POST/report-chat/manager-send — параметры: report_id, user_id, body (до 5000 символов)

6.2 Со стороны пользователя

Открывается отдельным модалом (data-form="report_chat_user"). При открытии сообщения от менеджера помечаются прочитанными.

POST/report-chat/user-send — параметры: report_id, body (до 5000 символов)

6.3 Визуальное оформление сообщений

ОтправительВыравниваниеСтиль
МенеджерСлеваСиняя левая граница, голубой фон
ПользовательСправаСерая правая граница, светло-серый фон

7. Счётчики непрочитанных сообщений

У пользователя — кнопка «Переписка»

Показывает количество непрочитанных сообщений от менеджера по данному отчёту. Сбрасывается при открытии чата.

CabinetMessageViewModel::unreadCountsByReportsForUser(
    array $reportIds,
    int   $userId
): array  // [report_id => count]

У менеджера — иконка чата

Показывает количество непрочитанных сообщений от пользователя по данному отчёту. Сбрасывается при открытии модала отчёта.

CabinetMessageViewModel::unreadCountsByReports(
    array  $reportIds,
    string $staffType,  // Manager::class
    int    $staffId
): array  // [report_id => count]
